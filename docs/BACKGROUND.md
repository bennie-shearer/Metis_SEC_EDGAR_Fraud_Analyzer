## Purpose

The Metis SEC EDGAR Fraud Analyzer applies quantitative forensic accounting
techniques to publicly available SEC EDGAR filings to help researchers, analysts,
and students identify potential financial statement irregularities. It is a
research and educational tool — not investment advice.

## Data Flow

```
Browser → GET /api/analyze?ticker=AAPL
  → HttpServer::handle_client()
    → FraudAnalyzer::analyze_by_ticker()
      → SECFetcher::lookup_company_by_ticker()   [SEC company_tickers.json]
      → SECFetcher::get_all_financial_data()     [SEC companyfacts/CIK*.json]
        → parse XBRL company facts for each filing
      → BeneishModel::calculate()
      → AltmanModel::calculate()
      → PiotroskiModel::calculate()
      → FraudTriangleModel::calculate()
      → BenfordModel::calculate()
      → detect_red_flags() / analyze_trends()
      → calculate_composite_score()
    → ResultExporter::to_json()
  → Cache::set()
← JSON response
```

## SEC EDGAR Integration

The analyzer uses two SEC EDGAR public APIs (no API key required, but a valid
User-Agent with email address is mandatory per SEC policy):

- `https://www.sec.gov/files/company_tickers.json` — ticker-to-CIK mapping
- `https://data.sec.gov/submissions/CIK{cik}.json` — filing history
- `https://data.sec.gov/api/xbrl/companyfacts/CIK{cik}.json` — XBRL financial data

Rate limiting: 100 ms between requests (configurable: `sec.rate_limit_ms`).

## GPU and Kubernetes Stubs

Two endpoints are reserved for future C++20 implementations:

- `GET /api/gpu/status` — placeholder for a future SYCL/CUDA parallel
  computation backend (Benford distribution, matrix operations).
- `GET /api/k8s/status` — placeholder for Kubernetes deployment with
  Prometheus metrics, readiness/liveness probes, and horizontal scaling.

Both return informational JSON and are controlled by `gpu.enabled` and
`k8s.enabled` in config.pson.

## Zero External Dependencies

The server requires no third-party libraries:

| Concern | Implementation |
|---------|---------------|
| HTTP client (Windows) | WinHTTP (OS-provided) |
| HTTP client (Linux/macOS) | System `curl` via `popen()` |
| JSON parsing | Custom `json.h` (header-only) |
| PSON parsing | Custom `pson.h` (header-only) |
| Logging | Custom `logger.h` (header-only) |
| Caching | Custom `cache.h` (header-only) |
| Threading | C++20 `std::thread` |
